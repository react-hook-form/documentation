## \</> `FieldArray:` <TypeText>Component</TypeText>

Component based on the [`useFieldArray`](/docs/usefieldarray) hook, useful for working with dynamic field arrays as a headless/controlled component. All props are the same as `useFieldArray`, plus a `render` prop that receives the hook's return value.

```javascript
<FieldArray
  control={control}
  name="test"
  render={({ fields }) =>
    fields.map((field, index) => (
      <input key={field.id} {...register(`test.${index}.value`)} />
    ))
  }
/>
```

### Props

---

| Name               | Type                               | Required | Description                                                                                                                                                                                                                                                                                                                                                     |
| ------------------ | ---------------------------------- | -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `render`           | <TypeText>Function</TypeText>      | âś“        | Render prop function that receives the same object [`useFieldArray`](/docs/usefieldarray#return) returns (`fields`, `append`, `prepend`, `insert`, `remove`, `swap`, `move`, `update`, `replace`) and returns a React element.                                                                                                                                  |
| `name`             | <TypeText>string</TypeText>        | âś“        | Name of the field array. **Note:** Dynamic names are not supported.                                                                                                                                                                                                                                                                                             |
| `control`          | <TypeText>Object</TypeText>        |          | [`control`](/docs/useform/control) object provided by `useForm`. It's optional if you are using `FormProvider`.                                                                                                                                                                                                                                                 |
| `shouldUnregister` | <TypeText>boolean</TypeText>       |          | Whether Field Array will be unregistered after unmount.                                                                                                                                                                                                                                                                                                         |
| `disabled`         | <TypeText>boolean</TypeText>       |          | Disables the entire field array. When `true`, `fields` initializes as an empty array, all mutation methods (`append`, `prepend`, `insert`, `remove`, `swap`, `move`, `update`, `replace`) become no-ops, and the array is not registered with the form. Useful for conditionally enabling a field array in discriminated union form shapes.                     |
| `keyName`          | <TypeText>string = "id"</TypeText> |          | Name of the attribute with autogenerated identifier to use as the `key` prop. This prop is no longer required and will be removed in the next major version.                                                                                                                                                                                                    |
| `rules`            | <TypeText>Object</TypeText>        |          | The same validation `rules` API as for [`register`](/docs/useform/register), which includes: `required`, `minLength`, `maxLength`, `validate`. In case of validation error, the `root` property is appended to `formState.errors?.fieldArray?.root` of type [`FieldError`](/ts/#FieldError). **Important:** This is only applicable to **built-in** validation. |

### Return

---

The following table contains information about the object passed to the `render` prop function â€” it's identical to what [`useFieldArray`](/docs/usefieldarray#return) returns.

| Name      | Type                                                                                  | Description                                                                                                                                                                                                                                          |
| --------- | ------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `fields`  | <TypeText>object & \{ id: string \}</TypeText>                                        | This `object` contains the `defaultValue` and `key` for your component. When a field entry includes `disabled: true`, the `disabled` attribute is automatically propagated to the registered input.                                                  |
| `append`  | <TypeText>(obj: object \| object[], focusOptions) => void</TypeText>                  | Append input/inputs to the end of your fields and focus. The input value will be registered during this action. **Important:** append data is required and not partial.                                                                              |
| `prepend` | <TypeText>(obj: object \| object[], focusOptions) => void</TypeText>                  | Prepend input/inputs to the start of your fields and focus. The input value will be registered during this action. **Important:** prepend data is required and not partial.                                                                          |
| `insert`  | <TypeText>(index: number, value: object \| object[], focusOptions) => void</TypeText> | Insert input/inputs at particular position and focus. **Important:** insert data is required and not partial.                                                                                                                                        |
| `swap`    | <TypeText>(from: number, to: number) => void</TypeText>                               | Swap input/inputs position.                                                                                                                                                                                                                          |
| `move`    | <TypeText>(from: number, to: number) => void</TypeText>                               | Move input/inputs to another position.                                                                                                                                                                                                               |
| `update`  | <TypeText>(index: number, obj: object) => void</TypeText>                             | Update input/inputs at a particular position, updated fields will get unmounted and remounted. If this is not desired behavior, please use [`setValue`](/docs/useform/setvalue) API instead. **Important:** update data is required and not partial. |
| `replace` | <TypeText>(obj: object[]) => void</TypeText>                                          | Replace the entire field array values.                                                                                                                                                                                                               |
| `remove`  | <TypeText>(index?: number \| number[]) => void</TypeText>                             | Remove input/inputs at particular position, or remove all when no index provided.                                                                                                                                                                    |

<Admonition type="important" title="Rules">

- The same rules that apply to [`useFieldArray`](/docs/usefieldarray#rules) apply here â€” `field.id` (not `index`) must be used as the component `key`, and `shouldUnregister: true` is not supported.
- `render` is called on every render, so keep it free of side effects.

</Admonition>

##### Examples:

---

```tsx copy
import { useForm, FieldArray } from "react-hook-form"

type FormValues = {
  test: { value: string }[]
}

export default function App() {
  const { control, register, handleSubmit } = useForm<FormValues>({
    defaultValues: {
      test: [{ value: "" }],
    },
  })

  return (
    <form onSubmit={handleSubmit((data) => console.log(data))}>
      <FieldArray
        control={control}
        name="test"
        render={({ fields, append, remove }) => (
          <>
            {fields.map((field, index) => (
              <div key={field.id}>
                <input {...register(`test.${index}.value`)} />
                <button type="button" onClick={() => remove(index)}>
                  Delete
                </button>
              </div>
            ))}
            <button type="button" onClick={() => append({ value: "" })}>
              Append
            </button>
          </>
        )}
      />
      <input type="submit" />
    </form>
  )
}
```
